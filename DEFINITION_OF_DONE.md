# Definition of Done (DoD) Checklist

## 🎯 Bir feature/bug fix "DONE" bo'lganini qanday bilamiz?

### Code Quality
- [ ] Code locally tested
- [ ] Unit tests yozilgan (minimal 80% coverage)
- [ ] Code review passed
- [ ] No TypeScript/linting errors
- [ ] No console.logs, debuggers

### Testing
- [ ] Unit tests pass
- [ ] Integration tests pass
- [ ] Manual testing completed
- [ ] Edge cases handled
- [ ] No regressions

### Documentation
- [ ] README updated (agar kerak)
- [ ] API docs updated
- [ ] Code comments added (complex logic)
- [ ] Commit messages clear

### Performance & Security
- [ ] No N+1 queries
- [ ] Performance acceptable
- [ ] No security vulnerabilities
- [ ] Dependencies updated (agar kerak)

### Deployment
- [ ] Build passes
- [ ] Staging environment tested
- [ ] Related tickets linked
- [ ] Ready for production

### Communication
- [ ] Team informed
- [ ] Stakeholders updated
- [ ] Documentation shared
- [ ] Knowledge transfer (if needed)

---

## Junior Uchun: Minimal DoD

```
✅ Code yozildi
✅ Tests pass
✅ Code review passed
✅ PR merged
```

## Middle Uchun: Full DoD

```
✅ Butun DoD checklist
✅ Performance considered
✅ Security reviewed
✅ Documentation complete
```

## Senior Uchun: Extended DoD

```
✅ Full DoD
✅ Architecture aligned
✅ Team training done
✅ Future scalability considered
```
